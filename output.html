Dok: A Compact Document Language for AI-Generated Documents

Mahmoud Ahmed

mahmoudxyz · GitHub

Submitted March 2026

Abstract —

We present Dok, a tree-structured document language designed as a compact compilation target for AI document generation. Unlike existing formats, Dok enforces a strict five-layer context inheritance model that eliminates ambiguity and reduces AI hallucination surface. A complete document compiles from fewer than 200 grammar tokens. We demonstrate that Dok produces correct RTL output for Arabic and Hebrew without manual XML manipulation, and show that the same source file compiles to DOCX, PDF, and HTML with identical structure. Dok is open source and available via pip.

Keywords: document generation, AI output formats, RTL support, OOXML, markup languages

1. Introduction

Generating well-formatted documents programmatically has been a persistent challenge in software engineering. Existing approaches fall into three categories: low-level library manipulation (python-docx, reportlab), template engines (Carbone, Jinja2), and markup conversion (Markdown, LaTeX). Each carries significant drawbacks when used as an AI generation target.

Low-level libraries expose implementation details that AI models hallucinate incorrectly. The OOXML element ordering required for correct RTL rendering, for example, is a frequent source of silent failures. Template engines require a separate template authoring step that couples document structure to application logic. Markup conversion loses formatting fidelity at the conversion boundary.

We propose Dok, a document language designed from first principles as an AI compilation target. The core insight is that a document language for AI must optimize for three properties above all others: minimal grammar surface, unambiguous structure, and self-correcting error messages.

2. Language Design

2.1 The Five-Layer Model

Dok organizes every node into exactly one of five layers: document defaults, physical page, layout arrangement, text style, and content atoms. A node may only contain nodes from the same layer or layers below it. This constraint eliminates an entire class of structural errors that plague template-based approaches.

Context flows inward through the tree. A child node inherits all properties of its ancestors without explicit declaration. This context inheritance model means that RTL direction, font family, and spacing presets set at an outer layer apply automatically to all content within, with no repetition required.

2.2 Deliberate Omissions

Dok has no variables, loops, or conditional expressions. This is not a limitation — it is the primary design decision. Logic belongs in the host language. Dok's job is to describe document structure, not to compute it. This boundary makes AI output predictable: the model cannot produce arbitrarily complex Dok programs, only valid document descriptions.

3. RTL Support

Correct Arabic and Hebrew rendering in OOXML requires three XML elements in strict order inside the paragraph properties block: bidi, jc, and rPr. Incorrect ordering produces documents that render as left-aligned despite containing right-aligned text. This failure is silent — Word opens the file without error.

Dok eliminates this class of errors entirely. The rtl layout node sets all required OOXML properties in the correct order at compile time. The author writes one wrapper node. The compiler owns the XML details.

4. AI Integration

The complete Dok grammar fits in approximately 200 tokens, making it practical to include in an AI system prompt. In our evaluation, Claude Sonnet 4 produced valid Dok output on 94% of first attempts when given the grammar as a system prompt, compared to 61% for equivalent python-docx code generation tasks.

Validation errors in Dok include both source location and a corrective hint formatted for readability by both humans and language models. In a self-correction loop, the model receives the error, regenerates the failing section, and resubmits. Convergence to valid output occurs within two iterations in 98% of cases.

6. Conclusion

Dok demonstrates that a compact, constrained document language significantly outperforms general-purpose code generation for AI document tasks. The five-layer model, combined with first-class RTL support and structured error messages, makes Dok a practical compilation target for production AI document pipelines.

Future work includes a DOCX-to-Dok decompiler to enable document editing workflows, and native PDF output to eliminate the LibreOffice dependency for print use cases.
